# External Integrations

**Analysis Date:** 2026-03-17

## APIs & External Services

**Sports Data & News:**
- **CBS Sports** - College basketball game scores and schedules
  - Endpoint: `https://www.cbssports.com/college-basketball/scoreboard/FBS/{YYYYMMDD}/`
  - Client: `requests` + BeautifulSoup4 parser
  - Auth: None (public HTML page)
  - Used by: `Python scripts/cbs_scraper.py` (Stage 1)
  - Fetches: Final game scores, team names, score differential

- **Google News RSS** - News articles for sentiment features
  - Endpoint: Google News RSS feed (no direct URL, accessed via feedparser)
  - Client: feedparser
  - Auth: None (public RSS)
  - Used by: `Python scripts/sentiment_features.py` (Cell 3, function: `fetch_google_news_articles()`)
  - Fetches: 20 RSS items; top 3 articles fetched in full for sentiment analysis

- **ESPN Sports API** - Team news, injuries, lineup changes
  - Endpoint: ESPN search/web endpoints (accessed via requests)
  - Client: requests + BeautifulSoup4
  - Auth: None (public web pages)
  - Used by: `Python scripts/sentiment_features.py`
    - Function: `fetch_espn_news_articles()` - 15 general team search results
    - Function: `fetch_espn_targeted()` - 10 results for "{team} injuries" suffix
    - Function: `fetch_espn_targeted()` - 10 results for "{team} lineup" suffix
  - Fetches: Team-specific news articles for sentiment/context features

- **Bart Torvik T-Rank (KenPom Ratings)** - Efficiency ratings and rankings
  - Endpoint: Multiple fallback sources in priority order
    1. `https://barttorvik.com/trank.php?json=1&year=2026` (JSON, preferred)
    2. `https://barttorvik.com/2026_team_results.json` (JSON alternate)
    3. `https://barttorvik.com/trankwebb_2026.json` (JSON alternate)
    4. `https://barttorvik.com/trank.php?csv=1&year=2026` (CSV fallback)
  - Client: requests, JSON/CSV parsing
  - Auth: None (public data, no login required)
  - Used by: `Python scripts/kenpom_ratings.py` (Stage 4)
  - Fetches: Adjusted efficiency metrics (adj_em, adj_o, adj_d), rankings for all 365 D-I teams
  - Validation: adj_em range [−40, +50], adj_o/adj_d range [60, 150]

**Vegas Odds:**
- **The Odds API** - Real-time Vegas moneylines and spreads
  - Endpoint: `https://api.the-odds-api.com/v4/sports/basketball_ncaab/odds/`
  - Client: requests
  - Auth: API key via `ODDS_API_KEY` environment variable
  - Used by: `Python scripts/odds_features.py` (Step 1.5, called from `predict_today.py` Step 7)
  - Parameters: `markets=h2h,spreads` (moneylines and spreads), `regions=us`, `oddsFormat=american`
  - Fetches: Moneyline odds and point spreads for all scheduled games
  - Bookmakers: Averages across preferred books (DraftKings, FanDuel, BetMGM, Caesars); falls back to all if none present
  - Quota: Logs `x-requests-remaining` header for monitoring; graceful degrade on 422/429 (quota exceeded)
  - Graceful fallback: If `ODDS_API_KEY` empty or API fails, returns empty list and skips odds enrichment
  - Line-move tracking: Compares current odds to `odds_snapshot.json` (snapshot from previous run); outputs movement strings (e.g., "−145 → −160")

**Efficiency Metrics (Sports-Reference):**
- **Sports Reference (basketball-reference.com)** - Team efficiency statistics
  - Endpoint: Team season pages (scraped dynamically)
  - Client: requests + BeautifulSoup4
  - Auth: None (public)
  - Used by: `Python scripts/efficiency_metrics.py` (Stage 3)
  - Fetches: eFG%, TOV%, ORB%, FTR for offensive and defensive stats
  - Fallback: If primary Bart Torvik source fails, secondary web scrape from Sports Reference

  - `Python scripts/model_cache/nn_classifier.keras` - TensorFlow classifier model
  - `Python scripts/model_cache/metadata.joblib` - Feature names, numeric columns, ensemble weights (schema v2.0)

- **Google Drive** - Notebook pipeline only (not used in standalone)
  - `/content/drive/MyDrive/ncaab_model_cache/` - Models saved from Colab notebooks

**Caching:**
- None external; local cache via GitHub Actions `actions/cache@v4`

  - Pip dependency cache: `.cache/pip` on GitHub Actions runner

## CI/CD & Deployment

**Hosting:**
- **Vercel** - Static site hosting for `website/index.html` and `website/predictions_latest.json`
  - SPA (Single Page Application) configured with rewrites in `website/vercel.json`
  - Cache-Control header: 1-hour server cache with stale-while-revalidate

**CI Pipeline:**
- **GitHub Actions** - Daily scheduled workflow (`.github/workflows/daily_predictions.yml`)
  - Trigger: Cron schedule (Mon-Fri 3pm EDT, Sat-Sun 10am EDT) + manual `workflow_dispatch`
  - Runner: ubuntu-latest
  - Timeout: 75 minutes
  - Steps:
    1. Checkout repo (v4)
    2. Determine training day (odd day-of-month → training enabled)
    3. Restore model cache from previous runs
    4. Set up Python 3.9
    5. Cache pip dependencies
    6. Install dependencies
    7–11. Conditional steps (if training_day):
       - Scrape CBS games (`cbs_scraper.py`)
       - Collect sentiment features (`sentiment_features.py`)
       - Collect efficiency metrics (`efficiency_metrics.py`)
       - Collect KenPom ratings (`kenpom_ratings.py`)
       - Assemble training data (`feature_assembly.py`)
       - Train models (`model_training.py`)
       - Save model cache
    12. Generate daily predictions (`predict_today.py --export-json`)
    13. Generate bracket predictions (`march_madness_bracket.py --export-json --skip-refresh`)
    14. Commit and push predictions to repo (via git config)

**Deployment:**
- Git push to main branch triggers Vercel auto-deploy
- `website/predictions_latest.json` and `website/bracket_predictions_2026.json` committed by workflow
- Vercel detects changes and rebuilds/caches static assets

## API Rate Limiting & Timeouts

**CBS Sports:**
- Politeness sleep: 1.5 seconds between page requests
- Request timeout: 20 seconds
- Retry attempts: 3 with backoff [2s, 5s, 10s]

**ESPN:**
- Rate limit sleep: 0.4 seconds between team fetches
- Request timeout: 11 seconds
- No explicit retry logic (individual article fetches fail silently)

**Bart Torvik T-Rank:**
- Request timeout: 20 seconds
- Retry attempts: 3 with backoff [2s, 5s, 10s]
- Multiple endpoint fallback strategy (4 URLs, JSON preferred)

**The Odds API:**
- Request timeout: 15 seconds
- Retry attempts: 2 with 2-second backoff
- Quota monitoring: logs `x-requests-remaining` header
- Graceful degrade on HTTP 422/429 (quota/validation error)

**Google News RSS / feedparser:**
- Request timeout: 11 seconds (via requests)
- No explicit retry in feedparser integration
